# Проект "Виджет банковских операций клиента"
## Описание:
Это виджет, который показывает несколько последних успешных банковских операций клиента.
## Установка:
* Клонируйте репозиторий:
  ```
  https://github.com/kveremyeva/homework_python.git
  ```
* Установите `poetry`
  ```
  pip install poetry
  ```
  Если `poetry` установлен, установите зависимости:
  ```
  # для первичной установки
  poetry install
  # для обновления
  poetry update
  ```
  Для просмотра установленных зависимостей используйте `poetry show --tree`
## Как работать
На данный момент реализованы различные модули, многие из них уже протестированы.
### Модуль `masks`
* `get_mask_card_number`, которая принимает на вход номер карты и возвращает ее маску, в которой видны первые 6 цифр и последние 4 цифры, остальные символы отображаются звездочками, номер разбит по блокам по 4 цифры, разделенным пробелами. Пример работы функции:
  ```
  7000792289606361     # входной аргумент
  7000 79** **** 6361  # выход функции
  ```
* `get_mask_account`, которая принимает на вход номер счета и возвращает его маску, видны только последние 4 цифры номера, а перед ними — две звездочки. Пример работы функции:
  ```
  73654108430135874305  # входной аргумент
  **4305  # выход функции
  ```
  
### Модуль `widget` 
* `mask_account_card`, которая умеет обрабатывать информацию о картах, и счетах, используется маскировка описанная выше.
* `get_date`,  которая принимает на вход строку с датой в формате "2024-03-11T02:26:18.671407" и возвращает строку с датой в формате "ДД.ММ.ГГГГ" ("11.03.2024") Пример работы функции:
   ```
  2024-03-11T02:26:18.671407  # входной аргумент
  11.03.2024  # выход функции
  ``` 
### Модуль `processing`  
* `filter_by_state`, которая принимает список словарей и опционально значение для ключа `state` (по умолчанию 'EXECUTED').
  Функция возвращает новый список словарей, содержащий только те словари, у которых ключ `state` соответствует указанному значению.
* `sort_by_date`, которая принимает список словарей и необязательный параметр, задающий порядок сортировки (по умолчанию — убывание). Возвращает новый список, отсортированный по дате (`date`).

### Модуль `generators`
* `filter_by_currency` фильтрует транзакции по заданной валюте (например USD).
* `transaction_description(transactions)` генератор, возвращающий описания транзакций.
* `card_number_generator(start, stop)` генератор номеров банковских карт в заданном диапазоне в формате "XXXX XXXX XXXX XXXX", где Х - цифры карты.

### Модуль `decorators`
* `log` добавлен декоратора который автоматически логирует начало и конец выполнения функции, а также ее результаты или возникшие ошибки.
* `my_function` параметр с которым работает декоратор `log` 

## Тестирование программы

Для всех модулей добавлены тест-кейсы:
* В `test_masks` были добавлены тесты для функций `get_mask_card_number` - проверяет правильность маскировки номера карты, `get_mask_account` - проверяет правильность маскирования номера счета.
* В `test_widget` были добавлены тесты для функций `mask_account_card` - корректно распознает и применяет нужный тип маскировки в зависимости от типа входных данных (карта или счет). и `get_date` проверяет правильность преобразования даты.
* В `test_processing` были добавлены тесты для функций `filter_by_state` - проверяет фильтрацию списка словарей по заданному статусу `state`, `sort_by_date` проверят корректность сортировки по дате.
* В `test_generators` были добавлены тесты для функций `filter_by_currency` - проверят корректность фильтрации по заданной валюте, `transaction_descriptions` - проверят, что функция возвращает корректные описания для каждой транзакции, и `card_number_generator` проверяют, что генератор выдает правильные номера карт в заданном диапазоне. 
* В `test_decorators` были написаны тесты, проверяющие функциональность декоратора.
